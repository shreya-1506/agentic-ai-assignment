# Employee Claims Portal Enhancement Request

You are working on an existing insurance claims application. Analyze the current codebase, identify the relevant frontend, backend, workflow, and AI extraction components, and implement the following enhancements while maintaining existing functionality.

## Objective

Improve the Employee Claim Filing experience by:

1. Replacing employee selection dropdown with employee number entry.
2. Moving document upload ahead of the information review step.
3. Automatically populating claim information from uploaded documents.
4. Converting the information page into a confirmation/review screen instead of a data entry form.

---

## Enhancement 1: Employee Number Input

### Current Behavior

Users select an employee from a dropdown when filing a claim.

### Required Change

Replace the dropdown with an Employee Number input field.

### Requirements

* Remove employee selection dropdown UI.
* Add employee number text input field.
* Validate employee number format.
* On entry, retrieve employee record from existing employee service/database.
* Display appropriate error states for:

  * Invalid employee number
  * Employee not found
  * Service unavailable
* Store employee information in the claim context/workflow state.

### Acceptance Criteria

* Employee lookup is performed using employee number.
* User can proceed only after successful employee validation.
* Existing claim workflow continues to function.

---

## Enhancement 2: Reorder Claim Filing Workflow

### Current Flow

Step 1 → Employee Information Form

Step 2 → Document Upload

Step 3 → Claim Submission

### New Flow

Step 1 → Employee Number Entry

Step 2 → Document Upload

Step 3 → AI Extraction Processing

Step 4 → Information Confirmation

Step 5 → Claim Submission

### Requirements

* Update routing/navigation.
* Update progress indicators.
* Preserve workflow state between steps.
* Ensure uploaded documents are available to extraction services before review screen loads.

### Acceptance Criteria

* Information confirmation screen is only shown after document upload and extraction processing complete.
* User cannot skip required steps.

---

## Enhancement 3: Auto-Populate Information from Uploaded Documents

### Objective

Leverage the existing document processing and extraction pipeline to pre-fill claim information automatically.

### Implementation Requirements

After document upload:

1. Trigger extraction workflow.
2. Parse all available extracted entities.
3. Populate claim data model.

Fields should include but are not limited to:

### Employee Information

* Employee Name
* Employee Number
* Employer

### Incident Information

* Date of Incident
* Time of Incident
* Location
* Description of Loss
* Cause of Loss

### Injury Information

* Injury Type
* Body Part Affected
* Severity

### Coverage Information

* Policy Number
* Coverage Type
* Coverage Status

### Additional Metadata

* Claim Type
* Supporting Documentation
* Any confidence scores produced by extraction models

### Error Handling

If extraction confidence is low:

* Flag field as "Needs Review"
* Highlight visually on confirmation screen

If data is unavailable:

* Display "Not Found"

### Acceptance Criteria

* Maximum available information is extracted automatically.
* Manual user entry should not be required under normal circumstances.
* Extracted information persists through claim submission.
